# JSON import and export

Taster can import tastes from a JSON file (admin, Import and export tab) and export the whole catalog back to the same format. The format is designed to be written by hand or generated by an AI assistant: give this page to your assistant along with your notes, and it can produce a ready-to-import file. Categories travel in their own file, documented at the end of this page.

## File envelope

```json
{
  "app": "taster",
  "version": 1,
  "tastes": []
}
```

Files are UTF-8. Exports start with a byte order mark (BOM) so Windows editors detect the encoding and display accents correctly, and the import accepts files with or without it. All three keys are required. `tastes` contains one object per taste. Before writing anything, the admin import shows a preview (entries to create, to update, and errors) and asks for confirmation; the same dry run is available directly on the API with `?dry=1` on the import endpoints. Import is per-item: valid entries are saved, invalid ones are reported with their index and an error code, and neither blocks the other. When an imported taste is created (not updated) and carries the `createdAt` field that exports write, that creation date is restored, so a full export then import on a fresh instance keeps the original timeline (default sort order, Atom feed dates).

## Taste fields

| Field | Type | Required | Notes |
| --- | --- | --- | --- |
| `id` | string (UUID) | no | Present in exports. A known `id` updates the existing taste (upsert); an unknown or missing `id` creates a new one. |
| `title` | string | **yes** | 1 to 300 characters. |
| `category` | string | **yes** | Category **slug** (stable, locale-independent: `movies`, `tv-shows`, `video-games`, `restaurants`, `books`, `music`), exact, or its display name, case-insensitive. Unknown categories are an error, never auto-created. |
| `rating` | integer 1-5 | no | 1 terrible, 2 bad, 3 okay, 4 good, 5 excellent. |
| `status` | string | no | A status name of that category, case-insensitive (e.g. `"Watched"`, `"Finished"` on an English instance). Unknown statuses are an error. |
| `favorite` | boolean | no | Defaults to `false`. |
| `published` | boolean | no | Defaults to `true`; `false` keeps the entry as a draft. |
| `tags` | string[] | no | Free-form labels (genres, cuisine, platform...). Max 50, deduplicated case-insensitively; commas are not allowed in a tag name (the list filter URL separates tags with them). |
| `date` | string | no | Flexible precision: `"1995"`, `"1995-03"` or `"1995-03-11"`. |
| `location` | object | no | `{ "lat": 45.76, "lng": 4.83 }`, both required together. Shown as map links on the taste page. |
| `externalReviewUrl` | string (URL) | no | "Read the full review" link, e.g. a blog post. http(s) only. |
| `sections` | object[] | no | The written review, see below. |
| `links` | object[] | no | Reference links, see below. |
| `image` | object | no | Embedded illustration, see below. |
| `imageFocus` | object | no | `{ "x": 0.5, "y": 0.2 }`, fractions of the image width and height between 0 and 1. Focal point that card thumbnails keep centered; absent means centered. |
| `createdAt` | string | no | Written by exports. Honored when the import creates the taste (restores the original creation date); ignored on updates. |
| `updatedAt` | string | no | Export metadata; ignored on import. |

### Sections

Each section is one part of the review. A single section without `subtitle` is a simple review; several sections split it into titled sub-parts, each with an optional sub-rating.

```json
{ "subtitle": "Soundtrack", "rating": 5, "text": "Markdown **works** here." }
```

- `text` supports Markdown, plus the spoiler syntax `||hidden text||` which renders as a click-to-reveal block.
- `subtitle` and `rating` are optional; fully empty sections are dropped.

### Links

```json
{ "label": "Wikipedia", "url": "https://en.wikipedia.org/wiki/Chrono_Trigger" }
```

Both keys required, http(s) URLs only.

### Image

```json
{ "mime": "image/webp", "base64": "UklGR..." }
```

- `mime` is `image/jpeg`, `image/png`, `image/webp`, `image/avif` or `image/gif`; `base64` is the raw file content, max 5 MB decoded.
- Every image is validated (magic bytes) then re-encoded server-side into two WebP variants (display + thumbnail); the original is never stored, so oversized sources are fine as long as they stay under 5 MB.
- Exports only embed images when requested (the "Include images" option); plain exports stay light.

## Error codes

`POST /api/admin/import` answers `{ "imported": n, "updated": n, "errors": [{ "index": i, "code": "..." }] }`. `index` is the 0-based position in `tastes`.

`TITLE_REQUIRED`, `CATEGORY_REQUIRED`, `CATEGORY_UNKNOWN`, `STATUS_UNKNOWN`, `INVALID_RATING`, `INVALID_DATE`, `INVALID_LOCATION`, `INVALID_URL`, `INVALID_IMAGE`, `IMAGE_TOO_LARGE`, `INVALID_SECTIONS`, `INVALID_LINKS`, `INVALID_TAGS`. A file that is not a Taster export at all answers `400 INVALID_FILE`.

## Minimal example

```json
{
  "app": "taster",
  "version": 1,
  "tastes": [
    { "title": "Chrono Trigger", "category": "video-games" }
  ]
}
```

## Complete example

```json
{
  "app": "taster",
  "version": 1,
  "tastes": [
    {
      "title": "Chrono Trigger",
      "category": "video-games",
      "rating": 5,
      "status": "Finished",
      "favorite": true,
      "published": true,
      "tags": ["JRPG", "SNES"],
      "date": "1995-03",
      "location": { "lat": 48.8584, "lng": 2.2945 },
      "externalReviewUrl": null,
      "sections": [
        {
          "subtitle": "Story",
          "rating": 5,
          "text": "Some **Markdown**, with ||a hidden spoiler|| when needed."
        },
        {
          "subtitle": "Soundtrack",
          "rating": 5,
          "text": "..."
        }
      ],
      "links": [
        { "label": "Wikipedia", "url": "https://en.wikipedia.org/wiki/Chrono_Trigger" }
      ],
      "image": { "mime": "image/webp", "base64": "UklGR..." }
    }
  ]
}
```

## Demo data

The repository ships ready-to-import sample files with fictional entries across all default categories: [data/taster-tastes-template-en.json](../data/taster-tastes-template-en.json) for an English-seeded instance (`SEED_LOCALE=en`, the default) and [data/taster-tastes-template-fr.json](../data/taster-tastes-template-fr.json) for a French one. Import one from the admin to see the app populated, then delete the entries or keep editing them. Status names must match your instance's categories, which is why there is one file per language; on an English instance, import [data/taster-categories-template-fr.json](../data/taster-categories-template-fr.json) first if you want the French demo set.

## Exports

- Full export: admin, Import and export tab, or `GET /api/admin/export` (`?images=1` to embed images).
- Single taste: `GET /api/admin/tastes/:id/export`.
- Output is pretty-printed with a stable key order (title, category, rating, status, favorite, published, tags, date, location, externalReviewUrl, sections, links, image) so the file stays pleasant to edit by hand; absent optional fields are omitted rather than set to null.
- Re-importing an export is idempotent: entries update in place thanks to their `id`.

## Categories file

Categories and their statuses are structure rather than content, so they use a separate file and separate endpoints: `GET /api/admin/export/categories` and `POST /api/admin/import/categories`, both also available from the Import and export tab. The repository ships [data/taster-categories-template-en.json](../data/taster-categories-template-en.json) and [data/taster-categories-template-fr.json](../data/taster-categories-template-fr.json), which recreate the default categories in each language.

```json
{
  "app": "taster",
  "version": 1,
  "categories": [
    {
      "slug": "movies",
      "name": "Movies",
      "icon": "film",
      "color": "#e35d6a",
      "statuses": ["Watched", "Watching", "To watch"]
    }
  ]
}
```

| Field | Type | Required | Notes |
| --- | --- | --- | --- |
| `slug` | string | no | Upsert key. A known slug updates that category; when absent, the category is matched by name instead. On creation, a well-formed free slug is kept, anything else is derived from the name. |
| `name` | string | **yes** | 1 to 100 characters. |
| `icon` | string | no | A Heroicons outline name; defaults to `tag`. |
| `color` | string | no | `#rrggbb`; defaults to `#8b5cf6`. |
| `statuses` | string[] | no | Ordered status names, max 30. When provided, the list replaces the category's statuses; a status whose name already exists keeps its identity, so tastes referencing it are untouched. When omitted, existing statuses are left alone (new categories get none). |

Import answers the same result shape as tastes, with the codes `NAME_REQUIRED` and `INVALID_STATUSES`; the whole file is applied in one transaction. Importing a categories file never deletes a category, so it is safe to run on a populated instance.
