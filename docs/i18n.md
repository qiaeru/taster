# Internationalization

The UI ships in French and English. **French is the source of truth** (`client/src/i18n/locales/fr.json`); English is the fallback for non-French browsers. The canonical locale list is `SUPPORTED_LOCALES` in `client/src/i18n/index.ts`; never hardcode a locale tuple anywhere else.

## How it works

- Flat-key JSON files, one per locale, with `{{placeholder}}` interpolation.
- Lookup chain: active locale, then French, then the raw key. A missing key is a bug; add every key to every locale in the same change.
- Only an explicit pick in the header persists (localStorage `taster:locale`) and wins on later visits; otherwise the instance's default language (admin Application tab) applies, falling back to browser detection. The active locale syncs `<html lang>`, and the picker lists every locale by its native name.
- Dates format through `Intl.DateTimeFormat` with the active locale, including partial dates (`1995`, `mars 1995` / `March 1995`, full dates).
- The PWA manifest is content-negotiated per `Accept-Language` on the server (`/manifest.webmanifest`).

## What is translated and what is not

UI strings only. Owner content (taste titles, reviews, tag/category/status names) is single-language user data and never enters the locale files. The star labels are UI strings. Server error codes are stable machine keys (`PASSWORD_TOO_SHORT`, `CATEGORY_IN_USE`...) translated client-side under keys like `password.error.PASSWORD_TOO_SHORT`.

## Adding a locale

1. Copy `en.json` or `fr.json` to the new locale file and translate every value (meaning over words).
2. Add the locale to `SUPPORTED_LOCALES` in `client/src/i18n/index.ts` and to the `Locale` type in `client/src/i18n/detect.ts`.
3. Add a manifest description and extend the `Accept-Language` negotiation (currently a hardcoded `fr`/`en` choice) in `server/src/routes/manifest.ts`.
4. Extend the `AppSettings.defaultLocale` union in `shared/src/models.ts`, the matching enum in `server/src/routes/admin-settings.ts` and the fr/en check in `server/src/lib/settings.ts`.
5. Consider a `data/taster-categories-template-<locale>.json` and a `data/taster-tastes-template-<locale>.json` (status names in demo files must match the category file's language).

## Key parity check

```bash
node -e "
const fr = Object.keys(require('./client/src/i18n/locales/fr.json'));
const en = Object.keys(require('./client/src/i18n/locales/en.json'));
const miss = (a, b, n) => a.filter((k) => !b.includes(k)).forEach((k) => console.log('missing in ' + n + ':', k));
miss(fr, en, 'en'); miss(en, fr, 'fr');
"
```
